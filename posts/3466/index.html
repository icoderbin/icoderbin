Android的resources.arsc文件结构

更新于2023-04-01|androidAndroid的resources.arsc文件结构

resources.arsc 文件是Android打包所产生的资源索引文件。在Android开发中，我们经常使用R.drawable.xxx、R.string.xxx 来获取图片、字符串资源，这背后就是通过arsc文件来实现的。R.drawable.xxx 等值会变成资源索引，资源索引是一个16进制的数字，这些数字存储在R.java文件中。通过资源索引，在arsc中找到资源的真正存储路径，然后再根据路径获取资源。例如getDrawable(R.drawable.img)在编译后成了getDrawable(2131099964)，再将id转为十六进制为2131099964 = 0x7f06013c。

资源索引的数据是有固定格式的，如下所示0xPPTTEEEE表示为 PackageId(2位) + TypeId（2位） + EntryId(4位)

  • PP:Package ID，包的命名空间，取值范围为[0x01, 0x7f]，第三方应用均为7f。
  • TT:资源类型，有anim、layout、mipmap、string、style等资源类型。
  • EEEE:代表某一类资源在偏移数组中的值。

而arsc文件可以简单理解为资源信息键值对的存储，资源索引为key，然后通过key找到找到资源名、资源存储路径等详细信息。

下面分析一下resources.arsc的文件结构，resources.arsc是一个二进制文件，文件的结构定义在ResourceTypes.h 文件中。可参考下面这张描述arsc文件结构的图。

整个文件可以分为三大块：

  • ResTableHeader : 文件头，用来定义文件类型、文件大小等数据。
  • ResStringPool : 资源项值字符串池，存放了 APK 中所有资源项值的字符串内容。
  • ResTablePackage ： 包含了头部信息、资源类型字符串常量池、资源项名称字符串常量池、多个子模块。

下面对这三个模块进行详解

ResTableHeader 文件头

如下为文件头包含的数据信息：

如下图所示为通过010Editor工具读取的resources.arsc文件头。

  • type 占2字节，用来表示该文件是个arsc文件。
  • headerSize 用来表示文件头的字节个数，如上所示为12，刚好 type+headerSize+size+packageCount 的尺寸为12.
  • size 表示整个文件的大小。
  • packageCount 表示后面ResTablePackage 结构中ResTable_package中的数量。后面可以看到数量一共是19个。

ResStringPool 字符串常量池

第二块存储的是字符串常量池，它存储了资源的所有字符串常量，如下所示为010Editor解析到的格式。

  • header部分为数据块头部信息，存储如下信息：
    • type: 这个数据块的类型，为字符串常量池。
    • headerSize: 数据块头部占用的尺寸。
    • size: 该数据块占用的尺寸。
  • stringCount: 存储字符串的数量。
  • styleCount: 存储字符串样式数量。
  • flags: 字符串池是通过utf-8编码还是按照顺序编码。 上图256表示为按照utf-8进行编码。
  • stringsStart: 字符串内容池的起始位置.
  • styleCount: 字符串样式池的起始位置。
  • stringoffsets: 所有字符串的偏移量。知道了偏移量，每个字符串所占用的空间都能确定了。
  • strdata: 所有的字符串

ResTablePackage

这个数据结构中包含了资源的详细信息，每一个包名包含一个ResTablePackage 结构。如下所示分析的arsc文件中有19个包名。

下面详细分析一下单个ResTablePackage 文件格式。主要包含以下五个部分：

  • ResTablePackageHeader : 头信息
  • typeStrings : 资源类型字符串池
  • keyStrings ： 资源项名称字符串池
  • ResTable_typeSpec ： 资源表规范
  • ResTable_type ： 资源表类型配置

头信息

下面为010Editor解析出来的头信息。

  • header 包含三个数据
    • type 标识头信息类型。
    • headerSize 表示头信息的大小
    • size 表示整个数据块的大小。
  • id 表示package id，这个是资源索引中0xPPTTEEEE 中PP的值。
  • name 表示包名
  • typeStrings 表示资源类型字符串池在 ResTablePackage 中的偏移量，typeStrings是资源类型的列表，可参考后续章节。
  • lastPublicType 表示一般资源类型字符串资源池的元素个数
  • keyStrings 表示资源名称字符串池在 ResTablePackage 中的偏移量
  • 一般指资源项名称字符串资源池的元素个数。

typeStrings

typeStrings 是资源类型字符串池，既然是资源类型，很容易就想到 string 、layout 、drawable 、mipmap 等等，这些都是资源类型。如下所示为解析到的资源类型格式。

  • header 同样是头信息，和前面的头信息记录的数据相同，Type表示类型，headerSize表示头大小，size表示数据块大小
  • stringCount 表示字符串的数量，如上图片中位19个。
  • styleCount 字符串类型数量
  • flags
  • stringsStart 字符串数组偏移位置
  • stylesStart 字符串类型数组偏移位置
  • stringoffsets 每个字符串偏移的位置。
  • strdata 为详细的字符串

keyStrings

这个数据结构和typeStrings 的数据结构是一样的，只不过保存的是资源名称字符串池，它包含了当前packageId下的所有资源的资源名称。

ResTable_typeSpec

ResTable_typeSpec 是资源表规范，用来描述资源项的配置差异性。系统根据不同设备的配置差异就可以加载不同的资源项。该部分数据结构对应结构体 ResTable_typeSpec

如下为解析到的数据：

  • header : ResChunkHeader，其 type 是 RES_TABLE_TYPE_SPEC_TYPE
  • id : 标识资源的 Type ID, Type ID 是指资源的类型 ID 。资源的类型有 animator、anim、color、drawable、layout、menu、raw、string 和 xml 等等若干种，每一种都会被赋予一个 ID
  • res0 : must be 0
  • res1 : must be 0
  • entryCount : 等于本类型的资源项个数,指名称相同的资源项的个数

紧接着后面的是 entryCount 个 uint_32 数组，数组每个元素都是用来描述资源项的配置差异性的。

ResTable_type

ResTable_type 是资源项的具体信息，包括资源项的名称，类型，值和配置等等。对应结构体 ResTable_type。

  • header : ResChunkHeader，其 type 是 RES_TABLE_TYPE_TYPE
  • id : 标识资源的 Type ID, Type ID 是指资源的类型 ID 。资源的类型有 animator、anim、color、drawable、layout、menu、raw、string 和 xml 等等若干种，每一种都会被赋予一个 ID
  • res0 : must be 0
  • res1 : must be 0
  • entryCount : 资源项的个数
  • entryStart ：资源项相对于本结构的偏移量
  • config : 资源的配置信息

config 之后是一个大小为 entryCount 的 uint32_t 数组，用于描述资源项数据库的偏移量。这个偏移量数组之后是一个 ResTable_entry[]，我们再来看一下这块内容。

ResTable_entry

  • size : 该结构体大小
  • flags : 标志位
  • key : 资源项名称在资源项名称字符串资源池的索引

根据 flags 的不同，后面的数据结构也有所不同。如果 flags 包含 FLAG_COMPLEX(0x0001)，则该数据结构是ResTable_map_entry，ResTable_map_entry 是继承自 ResTable_entry 的，在原有结构上多了两个 uint32_t 字段 parent 和 count。

  • parent 表示父资源项的 ID。
  • count 表示接下来有多少个 ResTable_map_entry。

  • dataType为各种资源类型，比如说TYPE_INT_BOOLEAN、TYPE_STRING等。
  • 具体的值，如果是TYPE_INT_BOOLEAN，如果是TYPE_STRING 则为字符串在字符串常量池中的索引。
